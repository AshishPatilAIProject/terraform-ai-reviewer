Business Impact:
Unencrypted buckets expose sensitive data to unauthorized access, increasing the risk of data breaches. Lack of public access controls leads to potential data leakage, compromising confidentiality and compliance. Absence of versioning can result in data loss or difficulty recovering from accidental deletions or modifications.

Recommended Action:
Enable encryption at rest on all S3 buckets to secure data. Implement strict public access policies to restrict unauthorized exposure. Activate versioning to facilitate data recovery and maintain data integrity. Regularly audit S3 configurations to ensure compliance with security best practices.

Remediation Plan

Finding: S3 Bucket Missing Encryption

Risk: Data stored in the S3 bucket is unencrypted at rest, increasing the risk of unauthorized access and data breaches.

Terraform Fix:
```hcl
resource "aws_s3_bucket" "example" {
bucket = "example-bucket"
}

resource "aws_s3_bucket_server_side_encryption_configuration" "example" {
bucket = aws_s3_bucket.example.id

rule {
apply_server_side_encryption_by_default {
sse_algorithm = "AES256"
}
}
}
```

---

Finding: S3 Bucket Without Versioning

Risk: Lack of versioning increases the risk of permanent data loss or accidental overwrites with no ability to recover previous versions.

Terraform Fix:
```hcl
resource "aws_s3_bucket" "example" {
bucket = "example-bucket"

versioning {
enabled = true
}
}
```

---

Finding: S3 Bucket Without Public Access Controls

Risk: The bucket is potentially exposed to public access, risking unauthorized data exposure or tampering.

Terraform Fix:
```hcl
resource "aws_s3_bucket_public_access_block" "example" {
bucket = aws_s3_bucket.example.id

block_public_acls = true
block_public_policy = true
ignore_public_acls = true
restrict_public_buckets = true
}
```
